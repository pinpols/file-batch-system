# 国际化（i18n）规范

> 适用范围：本仓所有后端 Java 代码（batch-common / batch-orchestrator / batch-trigger / batch-console-api / batch-worker-*）。
>
> 前端 console-ui 在独立仓库，本文 §6 列出后端约束 + 前端待办清单。

## 0. 一句话概览

后端 i18n 统一走 Spring `MessageSource` + `Accept-Language` 协商。

- **业务异常** → `BizException.of(code, key, args)`，`messages_<locale>.properties` 维护双语翻译，ExceptionHandler 链路按当前 Locale 翻译。
- **枚举字典** → `DictEnum.messageKey()` 默认返回 `dict.<simpleClassName>.<enumName>`，`ConsoleMetaQueryService` 按 Locale 解析。
- **默认 Locale** = `zh-CN`（前端不发 `Accept-Language` 时的 fallback）。
- **fallback 链**：`messageSource.getMessage(key, args, locale)` → 先 `messages_zh_CN.properties`（zh）/ `messages.properties`（默认）→ 缺 key 时回退到 `BizException` 的 literal message → `ResultCode.label()`（中文回退）。

## 2. 命名规范

### 2.1 业务错误 key

```
error.<scope>.<reason>
```

- `<scope>` 为业务域，使用单数 snake_case：`tenant` / `job` / `pipeline` / `workflow` / `excel` / `file` / `auth` / `quota` ...
- `<reason>` 为简短失败原因：`not_found` / `already_exists` / `invalid_format` / `state_conflict` / `required` ...
- 所有字母小写，分隔符用 `_`，禁用驼峰、连字符。

**示例**：

| key | en | zh-CN |
|---|---|---|
| `error.tenant.required` | `tenant is required` | `租户参数缺失` |
| `error.tenant.already_exists` | `tenant already exists: {0}` | `租户已存在:{0}` |
| `error.workflow.cancel_invalid_state` | `cannot cancel from {0}` | `当前状态 {0} 不可取消` |
| `error.field.iso_datetime_required` | `{0} must be ISO-8601 datetime` | `{0} 必须为 ISO-8601 日期时间格式` |

### 2.2 通用错误码 key（对应 `ResultCode` 枚举）

```
error.common.<lower_snake_resultcode>
```

- 对应 `ResultCode` 13 个枚举值：`error.common.success` / `error.common.invalid_argument` / `error.common.not_found` / ...
- 由 `BizMessageResolver.resolve(ResultCode, Locale)` 自动解析。

### 2.3 字典枚举 key

```
dict.<SimpleClassName>.<EnumName>
```

- `<SimpleClassName>` = `JobInstanceStatus` / `TriggerType` / `BatchType` ...
- `<EnumName>` = `RUNNING` / `SCHEDULED` / `IMPORT` ...
- 每个注册到 `ConsoleMetaQueryService.REGISTRATIONS` 的枚举都需要在 messages 文件里有对应 308 行（已自动 seed）。

**示例**：

| key | en | zh-CN |
|---|---|---|
| `dict.JobInstanceStatus.RUNNING` | `Running` | `执行中` |
| `dict.TriggerType.SCHEDULED` | `Scheduled` | `定时触发` |
| `dict.BatchType.IMPORT` | `Import` | `导入` |

### 2.4 占位符约定

- 用 `{0}` / `{1}` / `{2}` ...，与 `BizException.of(code, key, args)` 的 args 顺序一一对应。
- 占位符值会经 Java `MessageFormat` 渲染，**单引号** `'` 在 `MessageFormat` 中是转义字符，写翻译时需要写双引号 `''`（如 `It''s`）或避免用单引号。

## 3. 业务代码用法

### 3.1 抛业务异常

```java
// ✅ 推荐：i18n key + 占位符
throw BizException.of(ResultCode.NOT_FOUND, "error.tenant.not_found", tenantId);
throw BizException.of(ResultCode.CONFLICT, "error.job.code_already_exists", jobCode);

// ✅ 带 cause（资源 IO 失败、依赖异常等）
throw BizException.of(ResultCode.SYSTEM_ERROR, "error.excel.workbook_generate_failed", e, e.getMessage());

// ⚠️ 仅在以下场景用 (code, message) 老构造器:
//   1) 工具类签名签收 String message 参数（Guard.java / ConsoleQuerySupport 之类）
//   2) message 由上游调用方 / 业务逻辑动态计算且无法/不需要 i18n
//   3) 平台内部 panic-style 不会暴露给前端的错误
throw new BizException(ResultCode.NOT_FOUND, message);
```

### 3.2 添加新 key

1. 在 `messages.properties` 加英文翻译。
2. 在 `messages_zh_CN.properties` 加中文翻译（**两个文件 key 必须 1:1 对齐**，否则 zh 用户切 en 时会回退到中文）。
3. 在业务代码用 `BizException.of` 引用。
4. 提交。

### 3.3 ResultCode 通用文案

```java
@Autowired BizMessageResolver resolver;
String message = resolver.resolve(ResultCode.RATE_LIMITED);
// zh-CN → "请求过于频繁"
// en → "too many requests"
```

## 4. ExceptionHandler 集成

| Handler | 入口 | 翻译入口 |
|---|---|---|
| `AbstractApiExceptionHandler`（batch-common） | `@ExceptionHandler(BizException.class)` 等 | `bizMessageResolver.resolve(exception)` |
| `ConsoleApiExceptionHandler` | RestControllerAdvice | 同上（Setter 注入避免破坏构造器签名） |
| `TriggerApiExceptionHandler` | RestControllerAdvice | 同上 |

ExceptionHandler 不需要业务开发者关心 — 只需保证 throw 的是 `BizException.of(...)` 即可。

## 5. 测试

- `batch-common/src/test/java/io/github/pinpols/batch/common/i18n/BizMessageResolverTest` 覆盖 6 条路径：
  1. messageKey 命中资源（zh + en）
  2. messageKey 缺失 → 回退到 ResultCode.label()
  3. 老 literal 构造器 → 透传 message
  4. `resolve(ResultCode)` 通用错误码（zh + en）
  5. `resolve(ResultCode)` 资源不存在 → 回退到 `ResultCode.label()`
  6. null 输入安全
- 业务回归：console-api 380 单测 / orchestrator 490 / trigger 90 / worker-process 50 全过（i18n 迁移后）。
