## 系统性模式(跨模块)

1. **`@Transactional` 滥用在 Scheduler / @EventListener / Aspect** — 至少 6 处。CLAUDE.md 规则 #4 明确"只放 Service 公共方法",但全仓持续违反。需要 ArchUnit 守护。
2. **`Executors.new*` 直接 new ExecutorService** — orchestrator / trigger / console-api / worker-core 都有(累计 7+ 处)。没有 Spring 生命周期、没有 Actuator 指标、`CallerRunsPolicy` 阻塞调用线程。建议统一一个 `BatchAsyncConfiguration` 集中注册。
3. **无界 `ConcurrentHashMap` 内存泄漏** — orchestrator (`OrchestratorConfigCacheService.negativeCache`)、trigger (`warnedInvalidCron` / `inFlightFires`)、console-api (Excel session / summary cache)、worker-core (`consecutiveFailures`)。需要 Caffeine 替换。
4. **单实现接口形式主义** — trigger / orchestrator / worker-process / worker-dispatch 都有。可批量清理。
5. **跨 worker 模块重复代码** — `*StepBeanRegistrar` / `*RouteAdapter` / `*WorkerConfiguration` / Step setter 代码,本该抽到 worker-core。

| 顺序 | 范围 | 工程量 | 收益 |
