## R7 教训

1. **超时缺失是隐藏的雷区** —— JDK `HttpURLConnection` 默认无超时，本仓库有 3 处独立 RestClient 自构造路径（worker registry / console internal / file dispatch），只有 OkHttp / HttpTaskExecutionClient 显式设。统一在 `OrchestratorInternalRestClient` + `HttpWorkerRegistryClient` 加保守值才闭环。
2. **EXPLAIN 不是只读的** —— PG `EXPLAIN ANALYZE <DML>` 真执行 DML 是非常容易踩的问题；任何 `"EXPLAIN " + userInput` 都是潜在 SQL 注入向量。
3. **Helm 配置漂移要靠 ConfigMap 单一权威源** —— `.env.prod` / `values.yaml` / `templates/configmap.yaml` 三层任一遗漏即引入静默退化，需要 Helm pre-deploy lint 守护（与 R4 教训一致）。
4. **指标命名一旦发布到 Prometheus 就要锁死** —— `giveup` vs `give_up` 这种"看不出问题但聚合就漏"的命名漂移是分布式监控的常见死角；建议 ArchUnit 加 `batch.\\w+\\.give_up\\.total` 命名规范。
