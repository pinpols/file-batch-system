## 观察

- **TOP 2 模块仍是 console-api + orchestrator**,合计占 Java 总量 **57%**(5-29 时 67%)—— 占比下降是因为 SDK / atomic / e2e 新模块上线分薄。
- **`batch-worker-atomic` test/main = 1.00**:dual-use 风险隔离模块,测试覆盖等于实现行数,符合 ADR-029 安全前置约束。
- **`batch-worker-sdk` test/main = 1.41**:作为对外发布 jar,公共契约 + 失败模式矩阵 + JDK HttpServer stub 测试三件套堆出来的,合理。
- **`batch-worker-sdk-testkit` test/main = 0.17**:本身就是给租户测自己 handler 用的工具模块,自身测试少不奇怪;契约由 SDK 主模块覆盖。
- **`batch-e2e-tests` 全是 test**:独立 IT 模块,7,983 行覆盖 30 个 IT,平均 ~266 行/IT。
- **Python SDK 12.9k 行**:作为 Java SDK 的对等实现 + 跨 SDK contract drift guard(Lane P),工程量与 Java SDK 主模块相当。
- 各模块 `test/main` 比 → 适合补测的候选:
  - 偏低(≤ 0.50):`batch-console-api`(0.49)、`batch-worker-core`(0.46)、`batch-worker-export`(0.46)
  - 健康(0.50 ~ 0.75):`batch-common`、`batch-orchestrator`、`batch-worker-process`、`batch-worker-dispatch`、`batch-worker-import`、`batch-worker-sdk-starter`
  - 偏高但合理(≥ 0.85):`batch-trigger`(0.88,调度逻辑高敏)、`batch-worker-atomic`(1.00,安全隔离)、`batch-worker-sdk`(1.41,对外契约)
- **docs/ 73k 行 / 350 文件**:文档体量已超 1/3 代码;`docs/changelog.md` + `docs/archive/` 定期归档过时 design doc 仍是当前痛点,见 [`docs/backlog/`](../backlog/) 治理计划。
- **`.sql` 12.4k 行 / 202 文件**:Flyway migrations 已到 V166(平均 ~62 行/migration),与 5-29 时 V153 相比新增 13 个 migration。

## 复算

```bash
# 总行数(类型分布)
for ext in java py xml yml yaml sql sh md; do
  files=$(find . -type d \( -name target -o -name build -o -name logs -o -name .claude -o -name node_modules -o -name .idea -o -name .venv -o -name .git \) -prune -o -name "*.$ext" -type f -print 2>/dev/null | wc -l)
  lines=$(find . -type d \( -name target -o -name build -o -name logs -o -name .claude -o -name node_modules -o -name .idea -o -name .venv -o -name .git \) -prune -o -name "*.$ext" -type f -print 2>/dev/null | xargs cat 2>/dev/null | wc -l)
  printf ".%-6s  %6s files  %8s lines\n" "$ext" "$files" "$lines"
done

# Java 按模块(main+test)
for m in batch-common batch-trigger batch-orchestrator batch-worker-core \
         batch-worker-import batch-worker-export batch-worker-process \
         batch-worker-dispatch batch-worker-atomic \
         batch-worker-sdk batch-worker-sdk-spring-boot-starter batch-worker-sdk-testkit \
         batch-console-api batch-e2e-tests; do
  main_lines=$(find "$m/src/main/java" -name '*.java' -type f 2>/dev/null | xargs cat 2>/dev/null | wc -l)
  test_lines=$(find "$m/src/test/java" -name '*.java' -type f 2>/dev/null | xargs cat 2>/dev/null | wc -l)
  echo "$m  main=$main_lines  test=$test_lines"
done

# Python SDK
find batch-worker-sdk-python/src -name '*.py' -type f | xargs cat | wc -l
find batch-worker-sdk-python/tests -name '*.py' -type f | xargs cat | wc -l
```
