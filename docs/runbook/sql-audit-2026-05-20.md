## 2. 用户配置 SQL 约束体系

### 2.1 SQL 入口字段

| 字段 | 来源 | 用途 | 约束等级 |
|---|---|---|---|
| `defaultQuerySql` (FileTemplate) | Console Export 配置 | 导出查询 | **强** |
| `queryParamSchemaJson` (FileTemplate) | Console Export 配置 | 参数 schema | 中 |
| `stepParams.sourceSql` (SqlTransform) | Pipeline PROCESS | 业务源 SELECT | **强** |
| `stepParams.queryTemplate` (SqlExport) | Pipeline EXPORT | 模板 SQL | **强** |
| `stepParams.validationSql` (SqlTransform) | Pipeline VALIDATE | 行级校验 | **强** |
| pipeline_definition.step_params (通用 JSON) | Console 保存 | 透传 plugin | 弱 |

### 2.2 强约束三件套

Process/Export validator 共同保障:

1. **AST 校验**:JSqlParser 拒非 Select(CTAS 因解析为 `CreateTable` 自动拒绝)
2. **Schema allowlist**:默认 `["biz"]`,要求 `schema.table` 全限定,拒 pg_catalog 等系统 schema
3. **Named parameter 必填**:`:tenantId` + `:batchNo`(Export)、自定义参数白名单
4. **禁用 PG 函数子集**(`49a92504` 加):
   ```
   dblink, pg_terminate_backend, pg_cancel_backend,
   pg_read_server_files, pg_read_binary_file, pg_ls_dir,
   copy_from_program, lo_import, lo_export
   ```
5. **顶层 LIMIT 上限**(Process 可选,默认关):`maxLimitRows=100k`

### 2.3 已知遗留风险

| 风险 | 路径 | 缓解 |
|---|---|---|
| stepParams 通用 JSON 无全局预执行审计 | pipeline_definition.step_params | 各 plugin 自律(SqlTransformCompute / SqlTemplateExport) — 新 plugin 必须走 validator 注册 |
| Process LIMIT 强制 default false | `batch.worker.process.sql-transform.require-limit` | 历史 pipeline 可能未带 LIMIT,生产逐步切 true |
| `queryParamSchemaJson` 无关键字黑名单 | Console Export 配置 | 仅 JSON Schema 格式校 |

## 3. 判断标准 — SELECT * / OFFSET 改造 ROI

**不要全改**。判断标准:

- **必改**(估 10-20 处):
  - 含 TOAST 大字段的表:`job_instance` (stack_trace + params JSONB + result_summary)、`workflow_run` (payload + error_message)、`file_record` (metadata_json)
  - 高 QPS Console list / Dashboard 自动刷接口(P95 慢查询数据指向 SELECT *)
  - 批量 dispatch 热循环(每秒数千次的 full-entity 拉取)

- **不必改**(其余 140+):
  - insert 后 select、id 详情、internal join lookup — 已按需读
  - 维护成本翻倍:ResultMap 双份手工同步
  - 没有 P99 / 慢查询数据指向 SELECT * → premature optimization

**诊断 SQL — 找前 20 肥表平均行大小**:

```sql
SELECT relname,
       pg_size_pretty(pg_relation_size(oid) / (reltuples::numeric + 1)) avg_row
FROM pg_class
WHERE relkind = 'r'
  AND relnamespace = (SELECT oid FROM pg_namespace WHERE nspname = 'batch')
ORDER BY pg_relation_size(oid) / (reltuples::numeric + 1) DESC
LIMIT 20;
```

OFFSET → keyset 同源判断:小数据集 OFFSET 性能差异微乎其微,只治理 P1000+ 翻页接口。
